# Google Maps CN Satellite Corrector 谷歌地图卫星图纠偏

A Chrome / Edge (Manifest V3) extension that fixes the misalignment between
the **satellite imagery** and the **road/label layer** on
[google.com/maps](https://www.google.com/maps) inside mainland China.

修正谷歌地图在中国大陆地区卫星图层与道路/标注图层之间的位置偏移。

## Why the offset exists

Chinese regulations require map providers to publish street data in the
obfuscated **GCJ-02** ("Mars") coordinate system. On Google Maps the
road/label layer for the mainland is GCJ-02, but by default the satellite
imagery is true **WGS-84**, so the two layers disagree by roughly
**100–700 m** depending on location (e.g. ~540 m near Guangzhou).

## How this extension fixes it

Google already solves this server-side: when the page is loaded with the
**`gl=cn`** query parameter, Google serves satellite imagery that is shifted
to GCJ-02, so imagery, labels and markers all line up. (Approach from
[Cygra/google-map-cn-correction](https://github.com/Cygra/google-map-cn-correction).)

The extension is a single content script that runs before the Maps app
boots and adds `gl=cn` to the URL when it is missing. To keep the side
effects of `gl=cn` (China-region search bias) away from unrelated browsing,
URLs whose `/@lat,lng` viewport is outside mainland China — including
Taiwan / Hong Kong / Macau, where Google's labels are *not* GCJ-02
shifted — are left untouched.

Verified against the live site: with `gl=cn` the imagery over Guangzhou
shifts by exactly the local WGS-84→GCJ-02 delta (~124 px east, ~68 px down
at z15) and lines up with the label layer.

## Installation (load unpacked)

**Chrome:** open `chrome://extensions` → enable **Developer mode** →
**Load unpacked** → select this folder.

**Edge:** open `edge://extensions` → enable **Developer mode** →
**Load unpacked** → select this folder.

Then open the reference page and toggle the satellite layer:
<https://www.google.com/maps/@23.1232526,113.2375627,5600m/data=!3m1!1e3>
The page reloads once with `gl=cn` and roads/labels sit on top of the
matching imagery.

Use the toolbar popup to enable/disable the correction (reload the Maps tab
after changing it).

## Development

No build step; plain JS. Run the tests with Node (≥ 18):

```sh
node test/content.test.js
```

Icons are generated by `tools/make_icons.py` (requires Pillow).

## Files

| File | Purpose |
| --- | --- |
| `manifest.json` | MV3 manifest; injects the script at `document_start` |
| `content.js` | URL check + one-time redirect adding `gl=cn` |
| `popup.html` / `popup.js` | Toolbar popup with the enable toggle |
| `test/` | Node test suite |

## Notes & limitations

- `gl=cn` biases Maps search results toward the China region while active.
  The extension therefore only applies it when the URL's viewport is inside
  mainland China (or has no viewport coordinates yet).
- Versions 1.x did the correction client-side by intercepting and
  re-compositing satellite tiles. That approach fought Google's rendering
  pipeline (see git history) and was replaced by the server-side `gl=cn`
  parameter in 2.0.0.
